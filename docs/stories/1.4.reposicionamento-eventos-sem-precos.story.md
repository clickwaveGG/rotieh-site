# Story 1.4 — Reposicionamento eventos-first, remoção de preços e fluxo "Saber mais"

**Status:** Ready
**Épico:** 1 — LP Rotieh
**Origem:** Briefing direto do Leonardo (2026-07-23)

## Descrição

O cliente principal do Rotieh é quem aluga o espaço como um todo para casamentos,
aniversários e festas em geral. O site deve deixar isso claro em toda a narrativa,
mantendo em evidência o convite a casais que queiram alugar um chalé para dois.
Preços saem do ar: cada produto exibe apenas descrição + botão "Saber mais", que
leva a um pré-atendimento (questionário) antes do WhatsApp, para que o cliente
chegue à conversa com as informações já estruturadas.

## Acceptance Criteria

1. **Dado** um visitante no site, **quando** vê hero/intro/modalidades, **então**
   a mensagem dominante é festas/eventos (casamentos, aniversários, festas) e o
   chalé pra dois aparece como convite explícito secundário.
2. **Dado** qualquer seção do site público, **então** nenhum valor em R$ é exibido
   (modalidades, pousada, eventos, extras, FAQ, questionário). Painel /gestao demo
   não é afetado.
3. **Dado** um produto/modalidade, **quando** o visitante clica em "Saber mais",
   **então** é levado ao questionário (#reserva) com a modalidade já selecionada.
4. **Dado** o questionário preenchido, **quando** envia, **então** a mensagem
   estruturada abre no WhatsApp pedindo disponibilidade e valores.
5. Build de produção (`npm run build`) passa sem erros.

## Escopo

- **IN:** copy/hierarquia (Hero, BookingBar, Eventos, Precos, Pousada, FAQ, data.js),
  remoção de preços, pré-seleção de modalidade no Questionario, tipo de evento no
  pré-atendimento.
- **OUT:** backend, novas fotos, domínio, alterações no /gestao.
